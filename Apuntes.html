LENGUAJE HTML

Es el lenguaje de marcas que utilizamos para dotar de estructura al contenido de la Web. Las etiquetas dotan de significado semántico al texto que encierran, y que será interpretado por el navegador.

HTML no se preocupa de la presentación de los elementos, sino solo de su definición. De hecho, puede usarse un simple editor de texto. Para la presentación se usa otro lenguaje, el CSS. Así, todo lo que tenga que ver con la presentación deberemos modificarlo mediante CSS, preferiblemente.

1. ETIQUETAS

Las etiquetas son una forma de estructurar una página web y su contenido. La mayoría de las etiquetas aparecen de dos en dos, con una de inicio y otra de cierre, y siempre encerradas entre dos símbolos de desigualdad (menor que < y mayor que >).
Existen algunas etiquetas (que no llevan nunca contenido) que no van en parejas, sino que aparecen aisladas sin etiqueta de cierre. Por ejemplo: img, br(salto de linea).
Las etiquetas se pueden escribir con mayúsculas o con minúsculas, pero lo más común es escribirlas en minúsculas.
Las etiquetas, además de su nombre, pueden proporcionar más información sobre el elemento empleando atributos a continuación de su nombre. Estos atributos suelen aparecer como: atributo = "valor" o atributo = `valor´, y puede haber más de un atributo.
Una buena práctica a la hora de escribir nuestro código HTML es cuidar su presentación para mejorar la legibilidad. En tipografía esto se llama SANGRAR, y en informática INDENTAR el código.

2. ATRIBUTOS

Hay dos atributos que destacan por su versatilidad y porque se usan mucho para/p poder trabajar con CSS y JavaScript para facilitar la comunicación entre estos distintos lenguajes:

  • class : el valor asignado pueden tenerlo diferentes elementos de la página.
  • id : el valor asignado debe ser único, y no repetirse.

Una lista completa de atributos disponibles, junto con las etiquetas HTML en las que están permitidos y su descripción, puede encontrarse en la página de referencia de atributos HTML de la Fundación Mozilla: Mas atributos html

3. ESTRUCTURA DEL DOCUMENTO

3.1. TIPO DE DOCUMENTO

La estructura básica de un documento HTML bien construido debe comenzar con la declaración del tipo de documento (doctype), que es el medio por el que indicamos al navegador qué versión de HTML vamos a utilizar. Con HTML5, que es la versión que vamos a usar, solo hay que indicar:

<!DOCTYPE html> Hay otros tipos de valores para el tipo de documento, según la versión de HTML a utilizar.

3.2. CONTENIDO DE LA PÁGINA

A continuación, aparecerá la etiqueta html, que encierra todo el contenido de la página. La etiqueta de cierre < /html > siempre aparecerá al final del documento, y no debe escribirse nada tras ella.

3.3. HEAD Y BODY

Dentro del elemento html incluiremos, obligatoriamente, otros dos elementos "hijos": head y body.

  • head: es el primer elemento, tras la etiqueta de apertura de html. Se empleará para proporcionar información al navegador sobre la página (cómo se llama, cuáles son las palabras clave utilizadas, quién es su autor, con qué SW se ha creado, qué idioma utiliza, ...). Son los metadatos de la web, y no se mostrarán dentro del documento.
  • body: albergará el contenido visible del sitio.

4. ELEMENTOS

Las etiquetas de apertura y cierre con contenido entre ellas forman un ELEMENTO HTML.

Los elementos tienen dos variantes:

  • Elementos de bloque: marcan secciones más grandes, o bloques, dentro del documento. Ocupan (salvo que se modifique mediante CSS) el 100% del ancho de la página. Además, el navegador web coloca nuevas líneas alrededor de los elementos de bloque, para que aparezcan en su propia línea.
  • Elementos en línea: para trozos más pequeños, normalmente dentro de una línea. No podremos modificar las propiedades relacionadas con su tamaño, ya que será, siempre, el de su contenido.

4.1. PRINCIPALES ELEMENTOS DE BLOQUE

Normalmente, los hijos directos de body serán elementos estructurales que organizan la página. Todos ellos, veremos varios, son elementos de bloque.

  1. Párrafos.
    El elemento de bloque más empleado para contener texto, que permite definir párrafos. En su interior puede haber texto, otros elementos en línea (inline), pero no elementos de bloque. Si no modificamos su aspecto usando CSS, los navegadores los muestran añadiendo un margen superior e inferior que sirve para diferenciar unos párrafos de otros en texto escrito. Si deseamos introducir un salto de línea, sin crear un nuevo párrafo, podremos recurrir a la etiqueta vacía br.

  2. div
    Sirve para crear una agrupación dentro de la cual se pueden incluir otros elementos. Además se le puede poner un atributo id o class. Los usaremos cuando creemos cajas, en el apartado 5.

  3. Encabezados.
    HTML proporciona 6 niveles de encabezados o títulos: h1, h2, h3, h4, h5 y h6, de mayor a menor importancia.

  4. Listas.
    Tres tipos de listas: ordenadas ol, desordenadas ul y listas de definición dl. Todas las listas pueden mostrar otras listas dentro de alguno de sus elementos. Son las llamadas LISTAS ANIDADAS. Nótese dónde se cierra el elemento li que contiene otra lista. Ahí va un ejemplo (ojo con la lista de definición):

  5. Citas.

    En ocasiones nos puede resultar útil mostrar fragmentos de textos procedentes de otras fuentes. Cuando estos fragmentos no se limitan a unas pocas palabras, sino que forman un elemento de bloque (como un párrafo) emplearemos la etiqueta blockquote para identificarlos.

4.2. ELEMENTOS INLINE O EN LÍNEA

Suelen usarse principalmente para contener texto que no forma bloques, como caracteres o palabras aisladas.

  1. Enlaces.

    Son el elemento más innovador del lenguaje html. Mediante ellos podemos enlazar en un documento diferentes recursos al pulsar sobre ellos. Para crearlos, disponemos de la etiqueta a(anchor); el destino al que nos llevarán se lo indicamos empleando el atributo href. En su interior suele aparecer el texto del enlace, pero también puede haber imágenes y otros elementos de bloque.
    Quedaría algo así:

    <a href="https://www.educa.jcyl.es/es">Aquí va el texto que quiero que aparezca </a>

    Hay dos tipos de enlaces:
    • DIRECCIÓN ABSOLUTA: incluye la ruta completa al recurso (URL): protocolo, servidor y ruta. Ejemplo:

      <a href="http://test.site.es/sitio/contacto.html">Contacto</a>

    • DIRECCIÓN RELATIVA: indica la ruta al recurso desde la ubicación del elemento desde el que se enlaza. Es decir, podremos usarlas cuando el sitio al que queramos ir esté en el mismo servidor que nuestra página. Ventajas: puedo crearlo antes de disponer de un dominio, y es más corto.
      Ejemplo:

      <a href="sitio/contacto.html">Contacto</a>

      • Además, si deseamos que el enlace se abra en una ventana nueva, podemos indicarlo mediante el atributo: target="_blank":

      Contacto

      También podemos decir al navegador que, en vez de cargar el destino del enlace, lo descargue. Usaremos el atributo download.

  2. Enlaces internos

    En documentos de mucha extensión podemos crear enlaces a secciones dentro del mismo documento. Se crean utilizando el atributo id en los destinos de los enlaces y dando el valor del id del destino, precedido por el carácter almohadilla # al atributo href de los enlaces.

  3. Etiqueta span

    Este elemento es un contenedor en línea (ocupa lo que ocupe su contenido) genérico que usaremos cuando queramos definir un elemento sin dotarlo de significado semántico. Es similar al elemento div, pero aquel era de bloque y este es de línea. Se usa normalmente para acceder al elemento usando CSS y asignarle estilos diferenciados.

  4. Etiqueta em

    Para dar cierto énfasis a un elemento disponemos de las etiquetas em. Los navegadores suelen representar el texto que contiene en cursiva, como la etiqueta i, pero este comportamiento lo podremos modificar mediante CSS.

  5. Etiqueta strong

    Se emplea para indicar que su contenido queremos resaltarlo con mayor énfasis que la etiqueta em. Los navegadores suelen representarlo en negrita, al igual que la etiqueta b. Como todas las etiquetas HTML, no debemos emplearla con fines visuales, sino semánticos. Con CSS ajustaremos cómo queremos que se muestren estos elementos strong.

  6. Las citas

    Para pequeñas citas en línea que no crean nuevos párrafos, tenemos el elemento q. Simplemente, coloca "" en el texto y lo marca para el caso de que quisiéramos aplicarle algún formato en CSS.

4.3. ELEMENTOS INCRUSTADOS

  1. La etiqueta img

    El elemento img se emplea para insertar imágenes de forma sencilla en una página web.
    Mediante el atributo src se indica la ubicación de la imagen. Los tipos de imagen permitidos son: png, jpeg, gif, svg y WebP.
    También se recomienda el uso del atributo alt con un texto alternativo a la imagen cuando las imágenes no sean solo decorativas. De este modo, en caso de que la imagen no se cargue, aparecerá ese texto.
    Los atributos width y height pueden incluirse para que el navegador conozca las medidas en píxeles de la imagen; de este modo, puede reservar el espacio que ocupará la imagen antes de cargarla.
    El siguiente ejemplo mostrará una imagen, guardada en una carpeta alojada en la misma carpeta que mi archivo HTML:
    Se escribe así:
    <img src="images/imagen_html5.png" alt="Imagen de ejemplo" width="300" height="250">

  2. El elemento figure

    Se suele emplear como contenedor para añadir a las imágenes, tablas, diagramas, etc. un significado semántico y mostrar su título. Para ello, se usa, dentro del elemento figure, la etiqueta figcaption. Veremos un ejemplo en el apartado TABLAS.

  3. Objeto iframe

    Permite anidar, dentro de nuestra página HTML, un nuevo contenedor, en el que se carga otro documento HTML. Este documento puede estar en nuestro mismo servidor, o cargarse desde una dirección de internet.
    Por ejemplo: Se escribiría así: <iframe src="https://sernagrp.com/blog/html/" width="600" height="400">Si no lo puede cargar, aparecerá este texto. </iframe >

CONSIDERACIONES A TENER EN CUENTA EN CSS:

Las propiedades "width" y "height" se aplican por defecto al contenido. Vamos a ver cómo hacer que se aplique a la caja que contiene ese texto. Para ello, vamos a utilizar la propiedad "box-sizing", con dos valores:

  • content-box (por defecto):Toma como referencia el contenido, y se le van añadiendo márgenes y/o bordes.
  • border-box: Toma como referencia el borde, que tendrá un margen exterior y otro interior, que lo separe del contenido.
